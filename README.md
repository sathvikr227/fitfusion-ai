# FitFusion AI

An AI-powered fitness & nutrition web app that generates personalised workout plans, tracks meals and macros, and coaches your form live through your camera.

Built with **Next.js 16**, **Supabase**, **Groq (Llama 3.3)**, and **MediaPipe Pose Landmarker**.

---

## Features

### Core

- **AI Workout Plans** — personalised multi-week plans generated by Llama 3.3 based on goals, equipment, and onboarding data.
- **Meal Logging + AI Food Analysis** — log meals by text or image; the assistant estimates calories, protein, carbs, and fat.
- **Body Metrics** — track weight, BMI, body fat, and progress over time.
- **Dashboard & Analytics** — streaks, weekly burn/consumption charts, habit tracking, injury risk widget.
- **AI Assistant Chat** — Groq-backed RAG assistant that pulls from a fitness knowledge base.
- **Weekly Reports** — AI-generated reflection on the past 7 days with a goal suggestion for next week.
- **Adaptive Re-plan** — regenerates your plan based on how the previous week went.

### Real-time Pose Coach ⭐

The pose tracker uses MediaPipe Pose Landmarker (running entirely in the browser) plus a custom validation layer:

- **Exercise validator** — per-frame checks confirm the user is actually doing the selected exercise. A shoulder press with wrists below the shoulders is rejected rather than miscounted.
- **Form checks** — multi-angle rules detect knee cave, hip sag, back rounding, elbow drift, shallow depth, and more.
- **Voice coach** — uses the Web Speech API to announce every rep, call out live form corrections, and deliver encouragement during long holds (planks). Throttled so it never spams.
- **Session insights** — ROM %, tempo, rejected attempts, and a ranked form-error breakdown per session.
- **Groq-powered AI feedback** — at the end of every session, a 3-paragraph personalised review (what went well, the #1 fix, and a cue for next time).
- **12 supported exercises** — bicep curl, shoulder press, push-up, lateral raise, dumbbell row, squat, lunge, deadlift, glute bridge, calf raise, jumping jack, plank (isometric hold).

## Getting Started

### Prerequisites

- Node.js 20+
- A Supabase project (free tier is fine)
- A Groq API key ([console.groq.com](https://console.groq.com))

### 1. Install

```bash
cd web
npm install
```

### 2. Environment variables

Create `web/.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
GROQ_API_KEY=your-groq-key
OPENAI_API_KEY=your-openai-key   # optional, only used for food image analysis
```

### 3. Database

Run the SQL files in `web/supabase/migrations/` inside the Supabase SQL editor, in order:

1. `20260411_fitfusion_additions.sql` — core tables (water, moods, workout/meal execution, assistant history, RAG knowledge).
2. `20260421_pose_sessions.sql` — pose session history.

### 4. Dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Noteworthy Design Decisions

- **Pose detection runs entirely client-side** — frames never leave the browser. Only the session summary (reps, tempo, form errors) is sent to the server for AI feedback.
- **Exercise validators** guard rep counting, so "selected squat, did pushup" can't rack up junk reps.
- **RLS on every table** — each user only sees their own data, enforced at the database layer.
- **Pose lib is decoupled** (`src/lib/pose/`) — adding a new exercise is just adding one config entry with validators + form checks.

---

## Known Limitations / Roadmap

- Pose accuracy degrades if the user is out of frame or the lighting is poor (MediaPipe limitation).
- Form checks use single-side (right) landmarks — symmetry/left-right comparison is on the roadmap.
- No rate-limiting on Groq endpoints (fine for a class project; for production, add Upstash).
- No automated test suite yet.
